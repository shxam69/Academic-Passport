# API Contract — MVP

Base path: `/api/v1`
Auth: `Authorization: Bearer <access_token>` on all routes except `/auth/*`.
Full OpenAPI spec will be auto-generated by Springdoc/Swagger from the controllers — this doc is the design-time contract, not a doc to hand-maintain in parallel.

## Auth
| Method | Path | Role | Description |
|---|---|---|---|
| POST | `/auth/register` | public | Student self-registration (requires valid `collegeCode`). Rate-limited per IP. |
| POST | `/auth/verify-email` | public | Confirms OTP/link, sets `is_verified=true` |
| POST | `/auth/login` | public | Returns access + refresh token. Rate-limited: 3 failures / 15 min / account locks out. |
| POST | `/auth/refresh` | public | Exchanges valid refresh token for new access token |
| POST | `/auth/logout` | authenticated | Revokes refresh token |
| POST | `/auth/forgot-password` | public | Sends reset link/OTP if email exists (don't leak whether it exists — always 200) |
| POST | `/auth/reset-password` | public | Consumes reset token, sets new password, revokes all existing refresh tokens for that user |

## Student
| Method | Path | Role | Description |
|---|---|---|---|
| GET | `/students/me` | STUDENT | Own profile |
| POST | `/marksheets` | STUDENT | Upload PDF (multipart) for a given `semesterId` |
| GET | `/marksheets/{id}/ocr` | STUDENT (owner) | View OCR-extracted data for review |
| PATCH | `/marksheets/{id}/ocr` | STUDENT (owner) | Correct OCR values before submit |
| POST | `/marksheets/{id}/submit` | STUDENT (owner) | Submit for staff verification |
| PUT | `/marksheets/{id}/reupload` | STUDENT (owner) | Re-upload after rejection only — 409 if current status isn't REJECTED. Overwrites file, resets OCR, flips verification back to PENDING. |
| GET | `/students/me/timeline` | STUDENT | Full academic history (all verified semesters) |
| GET | `/marksheets/{id}/download` | STUDENT (owner) | Pre-signed URL for verified record PDF |
| POST | `/support-tickets` | STUDENT | Raise a support ticket |

## Staff
| Method | Path | Role | Description |
|---|---|---|---|
| GET | `/staff/queue` | STAFF | Pending verifications, own department |
| GET | `/marksheets/{id}` | STAFF | Full detail: source PDF link + OCR + student info |
| PATCH | `/marksheets/{id}/ocr` | STAFF | Correct OCR values (staff override) |
| POST | `/marksheets/{id}/approve` | STAFF | Approve, locks record |

## Conventions
- All list endpoints support `?page=&size=` (default size 20, max 100).
- Errors follow a single shape: `{ "timestamp", "status", "error", "message", "path" }` via `@ControllerAdvice` global exception handler.
- File uploads: max 10MB, PDF only — validated by actual magic bytes, not just file extension or declared content-type (an attacker can rename a .exe to .pdf; extension checks alone don't catch this). Virus-scanned synchronously before a `marksheets` row is created with `virus_scan_status = CLEAN`; infected files are rejected at upload time with a 422, no row created. Recommend self-hosted ClamAV as a Docker sidecar — free, no per-scan cost.
- OCR is triggered asynchronously after upload (don't block the upload response on OCR completion) — client polls `GET /marksheets/{id}/ocr` or receives a notification when ready.
- Pre-signed download URLs expire in 15 minutes.

## Authorization — every `{id}` route, no exceptions
Role check (`@PreAuthorize`) is necessary but not sufficient. Every endpoint taking a resource `{id}` MUST also verify **ownership/scope in the service layer**:
- STUDENT routes: `marksheet.student.userId == currentUser.id` — otherwise any logged-in student can read/edit another student's marksheet by guessing sequential IDs (IDOR).
- STAFF routes: `marksheet.student.department.id == staff.department.id` — otherwise staff in Department A can approve/reject Department B's records.
This check belongs in the service layer, not just the controller — a future internal call that bypasses the controller (e.g. from a batch job) must not silently skip it.

## Super Admin bootstrap
There is no `/auth/register` path for SUPER_ADMIN — by design, it can't be self-service. The first Super Admin account is created by a one-time Flyway seed migration or a CLI command run at deploy time, not through the API. Document this explicitly in the deployment runbook so you're not locked out of your own platform on day one.
